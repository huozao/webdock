## Couple Photo Storage

Photos uploaded by AliECS Couple Memory are stored on the laptop path configured by `HOST_PHOTO_STORAGE_DIR`, default `/var/lib/webdock/photo_storage`.

Runtime files:

```text
HOST_PHOTO_STORAGE_DIR=/var/lib/webdock/photo_storage
PHOTO_STORAGE_DIR=/app/photo_storage
API_TOKEN=<runtime secret only>
```

API contract:

```text
POST   /storage/photos       multipart field: file
GET    /storage/photos/{key}
DELETE /storage/photos/{key}
```

All three endpoints require `Authorization: Bearer ${API_TOKEN}`. Keys are generated by WebDock and must match the stored filename pattern; callers should treat keys as opaque.

Minimal local check:

```bash
source /opt/webdock/deploy/laptop/.env
curl -i -H "Authorization: Bearer ${API_TOKEN}" http://127.0.0.1:18000/storage/photos/not-found.png
```

The command should return 404 for a missing key. A 401 means the API token is wrong or missing; connection failure means WebDock or the tunnel is down.

## Immich Maintenance

Immich runs independently from the WebDock container. Default host directory:

```bash
~/immich-app
```

Health:

```bash
IMMICH_DIR=~/immich-app deploy/laptop/immich/check-immich-health.sh
```

Backup preflight:

```bash
IMMICH_DIR=~/immich-app deploy/laptop/immich/backup-immich-preflight.sh
```

Safe update:

```bash
IMMICH_DIR=~/immich-app deploy/laptop/immich/update-immich-safe.sh
```

The update script refuses to continue unless a recent Immich database backup exists. Media files still require a separate backup of `UPLOAD_LOCATION`.

## Restart

```bash
sudo systemctl restart webdock
```

After restart, the browser should reopen with the persisted profile. You may need to run:

```bash
source /opt/webdock/deploy/laptop/.env
curl -X POST http://127.0.0.1:18000/browser/attach \
  -H "Authorization: Bearer ${API_TOKEN}"
```

## Stop

```bash
sudo systemctl stop webdock
```
